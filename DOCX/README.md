# Визуализация графа — руководство для пользователя

Настольное приложение на базе **Electron**: вы задаёте граф (матрицей смежности или из файла), видите его на экране и запускаете учебные алгоритмы. Расчёты тяжёлых алгоритмов выполняются нативным модулем на C++.

---

## С чего начать

1. Установите зависимости и соберите модуль алгоритмов (из корня проекта): `npm install`, затем `npm run build-addon` (или сразу `npm start`, если в скрипте уже есть сборка).
2. Запуск: `npm start` — откроется окно с полем графа и панелью кнопок.

---

## Основные действия на панели

- **Создать граф** — открывается меню: можно ввести данные вручную или **загрузить текстовый файл** (`.txt`) через диалог выбора файла.
- **Очистить** — убрать текущий граф с экрана.
- **Опции** — список **алгоритмов и задач** (см. ниже).
- **Редактировать граф** — добавить/удалить вершины и рёбра через формы (номера вершин, вес ребра, при необходимости «в обе стороны»).

**Тип графа:** переключатель **неориентированный** / **ориентированный** задаёт, как интерпретируются рёбра при построении.

### При создании графа (дополнительно)

- **«НЕ произвольная нумерация»** — можно задать **свои имена вершин** (буквы, слова); они отображаются внутри кругов вместо номеров 0, 1, 2…
- **Multi mod** — особый текстовый формат: первая строка — короткие имена вершин (до 3 символов), далее строки ребра вида «начало конец вес»; удобно для разреженного описания.

Вручную обычно вводится **квадратная матрица смежности**: в ячейке \((i,j)\) — вес ребра из \(i\) в \(j\), ноль означает отсутствие ребра (как в подсказке в окне ввода).

## Работа на холсте (поле графа)

- **Перетаскивание фона** — сдвиг вида (панорама).
- **Колёсико мыши** — масштаб вокруг курсора (в строке состояния показывается приблизительный масштаб).
- **Перетаскивание вершины** — развести вершины для удобного чтения схемы.
- **Мини-карта** в углу — обзор всего графа; **двойной щелчок** по мини-карте — **центрировать** вид.
- **Правая кнопка по вершине:** без Shift — выделить вершину (далее **Delete** удаляет её); при зажатом **Shift** — выбрать две вершины подряд, между ними добавится **двунаправленное ребро** (удобный быстрый режим правки).
- **Ctrl + Shift + A** — добавить новую вершину (к графу уже должен быть построен хотя бы один граф).
- **Escape** — сбросить выделение и незавершённую пару для ребра.

## Меню «Опции» — что умеет программа

### Обход графа

- **BFS (поиск в ширину)** — обход от выбранной стартовой вершины «по уровням».
- **DFS (поиск в глубину)** — обход вглубь.

Результат обычно подсвечивается на графе; порядок обхода можно проследить по визуализации.

### Поиск пути

- **Кратчайший путь (Дейкстра)** — между двумя указанными вершинами при неотрицательных весах.
- **Задача коммивояжёра (TSP)** — замкнутый маршрут по всем вершинам с минимальной суммой весов; решается методом **ветвей и границ** (для больших графов время может заметно расти).

### Остовное дерево

- **MST, Краскал** и **MST, Прим** — построение **минимального остовного дерева** двумя классическими способами; на графе выделяется набор рёбер дерева.

### Эйлеровость

- **Проверить на эйлеровость** — сообщает, можно ли обойти все рёбра ровно по одному разу (с учётом типа графа и степеней вершин).
- **Найти эйлеров цикл** — если цикл существует, программа его находит и показывает.

### Транспортная задача

По кнопке открывается окно ввода: размеры \(m\) и \(n\), матрица **тарифов**, строка **запасов** и строка **спроса**. Строки с `#` в начале игнорируются. Если сумма запасов не равна сумме спроса, задача **приводится к сбалансированной** за счёт фиктивного пункта с нулевыми тарифами. Строится **двудольный граф** перевозок и показывается **оптимальный план** (минимум затрат). Есть кнопка с **примером** для быстрой проверки.

### Задача о назначениях

- **Венгерский алгоритм** — оптимальное назначение «строка ↔ столбец» по матрице текущего графа (один выбор в каждой строке и в каждом столбце). Можно выбрать **минимизацию** суммы (затраты) или **максимизацию** (эффективность). Дополнительно можно задать **подписи** для левой и правой доли (например, сотрудники и задачи) — они используются в подписях на схеме.

### Прочее

- **Проверить связность** — ответ, является ли граф **связным** (из любой вершины можно дойти до любой по рёбрам).

---

## Для кого это приложение

Подходит для **закрепления теории графов и дискретной оптимизации**: наглядно видны обходы, пути, остовы, эйлеровы циклы, а также постановки **транспортной задачи** и **назначений** с визуализацией результата.

---

## Если что-то не работает

- Сообщения об ошибках часто выводятся в **строке состояния** под кнопками.
- Если алгоритмы не запускаются, убедитесь, что нативный модуль собран без ошибок (`npm run build-addon`) и граф **уже построен** (сначала «Создать граф» → «Построить»).
